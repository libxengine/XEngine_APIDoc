函数

RTPProtocol_Packet_Packet

功能

打包一帧数据

语法

bool RTPProtocol_Packet_Packet(LPCXSTR lpszClientID, int nPType, LPCXSTR lpszMsgBuffer, int nMsgLen, STREAMMEDIA_RTPPROTOCOL_PACKET * **pppSt_RTPPacket, int* pInt_PacketCount, XSHOT wProfile = 0, uint32_t * **pppnListExtern = NULL, int nExternCount = 0)

参数

 参数.一：lpszClientID
  In/Out：In
  类型：常量字符指针
  可空：N
  意思：输入要操作的客户端句柄
 参数.二：nPType
  In/Out：In
  类型：整数型
  可空：N
  意思：输入打包的负载类型ID
 参数.三：lpszMsgBuffer
  In/Out：In
  类型：常量字符指针
  可空：N
  意思：输入要投递的缓冲区,缓冲区必须为一帧一帧数据.请确保数据的正确性
 参数.四：nMsgLen
  In/Out：In
  类型：整数型
  可空：N
  意思：要投递缓冲区的大小
 参数.五：pppSt_RTPPacket
  In/Out：Out
  类型：三级指针
  可空：N
  意思：当前NAL单元组装的RTP包,这个内存需要调用基础库的内存释放函数
 参数.六：pInt_PacketCount
  In/Out：Out
  类型：整数型指针
  可空：N
  意思：输出RTP包个数
 参数.七：wProfile
  In/Out：In
  类型：无符号短整数型
  可空：Y
  意思：输入扩展头自定义标识符
 参数.八：pppnListExtern
  In/Out：In
  类型：三级指针
  可空：Y
  意思：输入扩展数据,这个内存由用户管理
 参数.九：nExternCount
  In/Out：In
  类型：整数型
  可空：Y
  意思：输入扩展数据个数

返回值

类型：逻辑型
意思：是否成功

备注

支持264,265,AAC,OPUS负载类型

投递数据后会通过三级指针直接返回一个可发送的RTP包列表
你可以使用AVFrame_Frame_Parse* 相关函数来解析帧
